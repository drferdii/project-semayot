3.3 · Error Envelope Standar
{ "error": { "code": "VALIDATION_ERROR", "message": "Nama produk wajib diisi", "details": [ { "path": "name", "issue": "required" } ] } }

Bentuk Error

Selalu bungkus di error: { code, message, details? }. code = string machine-readable. message = Bahasa Indonesia untuk ditampilkan ke user.

200 OK — success
201 Created — resource baru
400 Bad Request — validasi Zod fail
401 Unauthorized — no session
403 Forbidden — RLS reject
404 Not Found
500 Internal Server Error

HTTP Status Codes

Standar REST. Middleware otomatis 401 untuk no-session. RLS violation dari Supabase = 403 di handler.
