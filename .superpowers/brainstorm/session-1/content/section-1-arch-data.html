Key invariants (tidak boleh dilanggar)

  • Public site TIDAK fetch Supabase — 100% baca static
  • Admin TIDAK baca lib/semayot/* — data sendiri
  • Middleware /admin/* wajib check session
  • RLS = source of truth permission (defense in depth)
  • No runtime coupling public ↔ admin

Tradeoffs yang kita accept

  • Menu di public vs admin bisa out-of-sync (manual)
  • Owner harus maintain 2 sources untuk business info
  • Initial setup lebih lama (DB + RLS + seed)
  • Two deploy surfaces instead of one
Section 2 · Data Model (Supabase Postgres + RLS)
profiles
id · full_name · role · is_active
→ FK auth.users
RLS: owner full + self read

Profiles

Extend auth.users. role: 'owner' | 'staff'. is_active untuk soft-disable staff.

menu_items
id · name · description · price_cents · category
photo_url · badge · is_active
RLS: owner CRUD + staff read active

Menu items

price_cents INT (bukan float). category enum. photo dari Supabase Storage.

transactions
id · staff_id · total_cents · paid_cents
change_cents (GENERATED) · note
RLS: staff insert + owner all + staff read own

Transactions (POS header)

change_cents = GENERATED column Postgres. payment_method default 'cash'.

transaction_items
id · transaction_id · menu_item_id
name_snapshot · price_cents_snapshot
quantity · subtotal_cents
RLS: ikut parent policy

Transaction items

Snapshot nama & harga saat transaksi. History akurat walau menu diedit.

expenses
id · category · amount_cents
description · incurred_by · incurred_at
RLS: owner only

Expenses

category enum: bahan | operasional | gaji | lain. Cash out tracking.

ai_summaries
id · period_start · period_end
summary_text · generated_at
RLS: owner only · UNIQUE(period)

AI summaries cache

Cache hasil AI per periode. Avoid regenerate ringkasan harian yang sama.

Key decisions:
  • price_cents INT — hindari floating-point bug di pembukuan
  • Snapshot name_snapshot & price_cents_snapshot di line items — history tetap akurat
  • change_cents sebagai GENERATED column — Postgres hitung, mustahil salah
  • No FK ke auth.users.role — pisah profiles, RLS pakai EXISTS query
  • Indexes: created_at (tx), transaction_id (items), incurred_at (expenses), category (menu)
