6.2 · Testing Strategy

Unit test

Zod schemas · money formatting (cents → "Rp 45.000") · snapshot logic (name/price snapshot) · change calculation · date range filter · Indonesian number formatting

Tool: Vitest (zero-config untuk Next.js)

Integration test

API routes dengan Supabase mocked (per-route test). Verifikasi: response shape, status code, RLS policy integration, error envelope consistency.

Tool: Vitest + mock Supabase client

Manual smoke

End-to-end happy path: login → catat transaksi → lihat di laporan → generate AI summary → salin SEO rec. Checklist per slice.

6.3 · Environment Variables & Secrets
.env.local (dev, gitignored) + Vercel env (prod)
Variable Scope Tujuan
NEXT_PUBLIC_SUPABASE_URLpublicURL project Supabase (aman di-expose ke client)
NEXT_PUBLIC_SUPABASE_ANON_KEYpublicAnon key (RLS enforce via JWT)
SUPABASE_SERVICE_ROLE_KEYserver onlyBypass RLS — HANYA untuk migration/admin script. JANGAN pernah dipakai di runtime request handler
OPENROUTER_API_KEYserver onlySama seperti chat-widget. Dipakai di /api/admin/ai/*
NEXT_PUBLIC_SITE_URLpublicURL public site (untuk SEO recommendations: og:url, canonical)
⚠️ Hard rules:
  • Service role key tidak pernah dipakai di request handler runtime. Hanya untuk SQL migration & seed script.
  • .env.local di-gitignore (sudah). Vercel env vars di-set via dashboard, tidak di-commit.
  • Anon key aman di-expose karena RLS enforce permission. Service role key bypass RLS = privilege escalation kalau bocor.

6.6 · Risiko yang OG flag (state dulu, lalu mitigasi)

Risiko

  • Data finansial sensitif — revenue, expenses, profit. Mitigasi: RLS, HTTPS only, no PII di log, backup Supabase otomatis.
  • Single point of failure — dr. Alyn satu-satunya owner. Mitigasi: backup owner account via Supabase, dokumentasi recovery process.
  • Free tier limit — Supabase free 500MB DB, OpenRouter free ada rate limit. Mitigasi: monitor usage bulanan, plan upgrade path jelas.
  • Cash reconciliation — sistem tidak bisa match otomatis dengan uang fisik di laci. Owner tetap rekonsiliasi manual tiap tutup.
  • Tanpa audit log — perubahan data (hapus menu, edit expense) tidak ada jejak historis. Trade-off: simplicity vs compliance. Owner-only delete jadi lebih aman.
  • Public ↔ admin out-of-sync — menu di public site statis, admin bisa update DB. Mitigasi: panel settings ingatkan owner untuk update file TS manual.

YAGNI (jangan dibangun)

  • Multi-restaurant / multi-tenant
  • Online ordering / customer-facing checkout
  • Payment gateway (cash only, by design)
  • Customer accounts / loyalty program
  • Email marketing / newsletter blast
  • Mobile app native
  • i18n (Bahasa Indonesia only)
  • Dark mode admin
  • 2FA (overkill untuk warung)
  • Receipt printer integration
  • Inventory management
  • Supplier management
  • Market intel (sudah di-skip)
